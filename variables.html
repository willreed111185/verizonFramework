Framework v4

Variables

These variables are used throughout the framework and can be utilized by a developer to exploit the built-in functionality

Latest Changes

xGroup
designates the assigned work group (Accounting, FSG, Commissions, etc) - do not change
vPassword
stores the active password as assigned in XML/CyberArk Subscript - can replicate to handle multiple credentials
vProductionMode
choose between "Run", "Dev" (cyberark), and "Prod" (cyberark). can be used to switch dependent logic within script
xShareDriveRoot
url path to the main folder for the automation
xUserName
assigned variable from the XML. upon project creation it is autopopulated with system id. can be changed

Variable List

vCyberArkUserName
passed to CyberArk via VBA script. will be used to query vault. can be replicated to query multiple.
vLogErrorDate
sent to Main from SubTask to be used in error logging
vLogErrorline
sent to Main from SubTask to be used in error logging
vLogErrorMsg
sent to Main from SubTask to be used in error logging
vLogFileWorking
path to XLSX log file on the local machine
vMainStatus
status of the main task. can be used to determine flow control or logging procedure
vPassword
stores the active password as assigned in XML/CyberArk Subscript - can replicate to handle multiple credentials
vProductionMode
choose between "Run","Dev" (cyberark),and "Prod" (cyberark). can be used to switch dependent logic within script
vStatusSubTask
status of the current subtask, passed back to main by error handling. can be used to determine flow control or logging procedure
vSubTaskName
name of the current subtask
vTaskName
name of the current task
vTimeStamp
time assigned with script is initiated. can be used for folder structure or for other instances as a unique identifier. should be used in conjunction with system(user) in the event of scaling to differentiate concurrent process runs
vWorkingFilePath
path to local directory created. input files are copied here at runtime for local use by BOT
xArchiveSharePath
archive path on share drive. input files are copied here at runtime for audit purpose/tracking
xCOELogPath
path to TXT log path located on share drive. written to over network to prevent loss of content
xCyberArkVault
the name of the vault assigned to script due to group assignment. assigned at project creation
xGroup
designates the assigned work group (Accounting, FSG, Commissions, etc) - do not change
xInputSharePath
share drive location where input files can be dropped and auto copied to local Working directory (to be later copied to Output archive on share drive
xLogFilePath
path to share drive location for the XLSX log. this log is accessable from the Working directory after it is auto copied by the framework.
xOutPutSharePath
this is the share drive path where the Working directory is copied at bot completeion. this is Zipped by the framework - can be changed by user if desired
xProductionMode
choose between "Run","Dev" (cyberark),and "Prod" (cyberark). can be used to switch dependent logic within script
xShareDriveRoot
url path to the main folder for the automation
xTaskKillPath
path to file used in Environmental/Housekeeping subtask to stop preceeding tasks which may impeed BOT success. This is localized for each project. can be changed to suit individual needs via .cmd file
xUserName
assigned variable from the XML. upon project creation it is autopopulated with system id. can be changed
