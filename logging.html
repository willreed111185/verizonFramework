Logging

Logging is important so that we can know what the bot has done, when it was done, and how it was done (was it successful?).

Logging General Info

The framework had two methods of logging built into it. It has a granular COE log, which is in a text format. The other is a business user-level log. This is in a spreadsheet format and intended to be delivered to the business user if necessary.

The COE log is intended to be written to over the network, and to contain information about the status of the bot through the stages of its production. This is communicated over the network because of the desire to maintain a persistent record.

The business-user level log is intended to note the status of the iterations of the bot, as well as its processing success or failures so that this can be conveyed to the interested parties. This is in a spreadsheet format so that the data can be sorted, filtered, and configured to meet the needs of the business.

There are tasks in place throughout the framework which will handle much of this for the developer. Each task module has flow-control, and these trigger variables which can be used to note the logs. The COE log is written to throughout the execution of script, using the logging feature within Automation Anywhere.
