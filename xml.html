XML General Info

The XML file is created when the framework is kicked off with a new project. This file is used to convey information to AA about constants which the script will need to run effectively by initializing variables and allowing for rapid scaling.

Let’s take a look at why we need this:

Initialization

If we were to program a username into a script for a system, when we move the BOT to a new environment we will need to change that information. If a URL for a website changes (but the site stays the same) we may want to be able to access that outside the script. If we have a list of email recipients, that list may change and we don’t necessarily want to reprogram our AA script to do so. If our input folder changes from one drive to another, be it because of environment changes or some other administrative needs, we don’t want to redevelop our AA files to accommodate that.

Scaling

The XML, as implemented in the framework, also allows for us to scale our processes. If we have an identical process which needs to run on two machines (box1) and (box2), we can simply rename the scripts to incorporate the names into the XML and the framework will know which file to open. Box1 will be assigned UserName1 and Box2 will be assigned UserName2 so long as the files are named accordingly. This allows for not only two, but virtually an unlimited number of BOTs to utilize the script without ever changing the AA files which were developed.
